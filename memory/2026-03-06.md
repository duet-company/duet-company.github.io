- PR #12: FAILING (entrypoint wrapper - syntax error)
- PR #13: FAILING (entrypoint syntax fix - Docker still parses -c incorrectly)

### Root Cause of Previous Failures
GitHub Actions service options with `--entrypoint bash -c "command"` format
is being parsed by Docker as:
```
docker create --entrypoint bash -c "command" ...
```

Where Docker interprets `-c` as the `--cpu-shares` flag instead of a bash argument,
causing this error:
```
invalid argument "rm -rf ..." for "-c, --cpu-shares" flag
```

### PR #14 Innovation
PR #14 removes the entrypoint override entirely and relies on tmpfs mounts
to ensure clean state. This is simpler and avoids Docker parsing issues.
