### 🔍 CI FAILURE ROOT CAUSE FOUND & FIXED!

**The Problem:**
The `pytest_runtest_makereport` hook in `tests/test_report_generator.py` was only handling the "call" phase of test execution. When tests were skipped during the "setup" phase (which is when most skips happen), the hook never saw them. This caused:

- CI failing with: "❌ Failed: 14 (100.0%)" when all 14 tests were actually SKIPPED

```
Test report output: "❌ Failed: 14 (100.0%)"
```

**The Fix:**
Updated `pytest_runtest_makereport` hook to handle all three test phases:

1. **setup phase**: Captures tests skipped during setup and setup errors
   - Checks for `Skipped` exception type
   - Records status as "skipped"

2. **call phase**: Captures normal test execution
   - Passed: `call.excinfo is None`
   - Failed: Any other exception
   - Skipped: `Skipped` exception type
   - Error: `Error` exception type

3. **teardown phase**: Captures teardown errors
   - Records any teardown exceptions as "error"

**Key Changes:**
- Changed from `if call.when == "call":` to handling all phases
- Added explicit check for `Skipped` exception type
- Added `_report_recorded` flag to avoid duplicate entries
- Added error message capture for better debugging

### 💡 Technical Notes

**Pytest Test Phases:**
1. **setup**: Runs before test execution
   - Fixtures are set up
   - Tests can be skipped here (e.g., missing dependencies)
   - Errors here are "setup errors"

2. **call**: The actual test function executes
   - Test passes if no exception
   - Test fails if assertion fails or exception raised
   - Test can be skipped programmatically

3. **teardown**: Runs after test execution
   - Fixtures are torn down
   - Errors here don't affect test result (but indicate problems)

**Why Skipped Tests Were Failing:**
- The old hook only looked at `call.when == "call"`
- Skipped tests skip during `setup`, never reach `call`
- So the hook never recorded them
- Later, when tests weren't in results, they were assumed failed

**The Fix Pattern:**
```python
# Before (only handled call phase)
def pytest_runtest_makereport(item, call):
    if call.when == "call":
        status = "passed" if call.excinfo is None else "failed"

# After (handles all phases)
def pytest_runtest_makereport(item, call):
    if call.when == "setup":
        # Check for skipped tests
        if call.excinfo.typename == "Skipped":
            status = "skipped"
    elif call.when == "call":
        # Check for passed/failed/skipped/error
    elif call.when == "teardown":
        # Check for teardown errors
```
